📋 1. Ringkasan Project

Latar Belakang

Divisi Umum PTPN IV mengelola data perjalanan dinas untuk seluruh karyawan meliputi tiket pesawat, hotel, kereta api, sewa mobil, per diem, dan transportasi lokal. Data ini sebelumnya dikelola secara manual menggunakan file Excel yang terpisah-pisah per vendor dan per kategori.

Tujuan

Membangun Dashboard Rekap Perjalanan Dinas berbasis web yang dapat:

No Tujuan Deskripsi
1 Input Data Manual Menyediakan form input yang mudah digunakan dengan auto-calculation untuk berbagai vendor
2 Upload Data Excel Mendukung upload file Excel yang sudah ada agar bisa diproses otomatis
3 Visualisasi Data Menampilkan ringkasan pengeluaran dalam bentuk dashboard, grafik, dan tabel
4 Export Laporan Menghasilkan file Excel standar yang siap digunakan untuk pelaporan

Teknologi yang Digunakan

Komponen Teknologi Keterangan
Frontend HTML + CSS + JavaScript Vanilla (tanpa framework)
Styling CSS Custom Properties Design system berbasis CSS Variables
Excel Processing SheetJS (xlsx) Library untuk baca/tulis file Excel
Database localStorage Penyimpanan lokal di browser (sementara)
Typography Inter + JetBrains Mono Google Fonts
Server Lokal Python http.server Port 8080

🔢 4. Rumus Auto-Calculation

Section 4 (Nominal & Tagihan) menggunakan rumus kalkulasi otomatis yang berbeda tergantung vendor yang dipilih. Saat user memilih vendor, tabel ringkasan rumus akan tampil di form.

4.1 Vendor US (Permata Andalan Wisata)

Field Tipe Rumus
Tagihan ✏️ Input Manual —
Service Fee ✏️ Input Manual —
DPP Service Fee ⚡ Auto = 11/12 × Service Fee
PPN 12% dari Tagihan ⚡ Auto = Service Fee × 11%
PPH 23 ⚡ Auto = 2% × Service Fee
Tagihan Service setelah Pajak ⚡ Auto = Service Fee + PPN 12% − PPH 23
Jumlah Tagihan ⚡ Auto = Tagihan + Service Fee + PPN 12%
Yang Dibayarkan ⚡ Auto = Jumlah Tagihan − PPH 23

4.2 Vendor FES (Forever Express Service)

Vendor FES memiliki field Jumlah (malam/unit) sebagai pengali untuk Tagihan dan Service Fee. Contoh: jika menginap 2 malam, maka Tagihan dan Service Fee per malam dikalikan 2.

Field Tipe Rumus
Tagihan (per unit) ✏️ Input Manual —
Service Fee (per unit) ✏️ Input Manual —
Jumlah (malam/unit) ✏️ Input Manual —
PPH 23 ⚡ Auto = 2% × (Service Fee × Jumlah)
Jumlah Tagihan ⚡ Auto = (Tagihan × Jumlah) + (Service Fee × Jumlah)
Yang Dibayarkan ⚡ Auto = Jumlah Tagihan − PPH 23

Contoh Perhitungan FES (dari Invoice)

Input Nilai Hasil Nilai
Tagihan per malam Rp 2.300.000 Total Tagihan (×2) Rp 4.600.000
Service Fee per malam Rp 115.000 Total Service Fee (×2) Rp 230.000
Jumlah 2 PPH 23 (2% × 230.000) Rp 4.600
Jumlah Tagihan Rp 4.830.000
Yang Dibayarkan Rp 4.825.400

4.3 Vendor Lainnya

Untuk vendor selain US dan FES, semua field diinput secara manual: Tagihan, Service Fee, dan Yang Dibayarkan.

🔄 6. Alur Kerja User

Skenario: Input Data DMRS (3 baris — Pesawat, Hotel, Kereta)

1
Buka halaman input.html
Jika ada data sebelumnya di localStorage, data otomatis dimuat.
2
Isi Entri Pertama (Pesawat)
Isi semua section: Cost Center, Nomor GL, Divisi, Vendor → US, Kategori → Pesawat, Tagihan, Service Fee, Uraian. Klik Simpan Entri.
3
Klik tombol (+) untuk entri baru
Semua field ter-copy otomatis dari entri pertama. Hanya perlu ganti: Kategori → Hotel, Tagihan, Service Fee.
4
Simpan entri kedua (Hotel)
Data tersimpan dan auto-save ke localStorage.
5
Ulangi untuk entri ketiga (Kereta)
Klik (+), ganti Kategori → Kereta, isi nominal baru, simpan.
6
Review di Preview Table
Cek semua 3 entri di tabel preview bawah. Edit jika ada yang salah.
7
Export ke Excel
Klik 📊 Export Excel → file .xlsx ter-download dengan 2 sheet.
💡 Tips Data otomatis tersimpan di browser setiap kali klik "Simpan Entri". Jadi meskipun Anda menutup browser dan membuka kembali esok hari, data tetap ada. Gunakan tombol 🗑️ Hapus Semua untuk membersihkan data setelah selesai export.
